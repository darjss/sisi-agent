\section{Хиймэл оюуны агентын ойлголт}

Сүүлийн үеийн хэлний том загварууд нь байгалийн хэлээр асуултад хариулах чадвар өндөр болсон боловч зөвхөн текст үүсгэх нь бодит системтэй харилцахад хүрэлцэхгүй. Ялангуяа академик порталын өгөгдөл нь тухайн мөчийн бодит төлөвөөс хамаарах тул загвар нь гаднын өгөгдөл унших, шаардлагатай хэрэгсэл дуудах, үр дүнг нэгтгэн тайлбарлах чадвартай байх шаардлагатай. Энэ хүрээнд ``AI agent'' гэдэг ойлголт нь хэлний загвар, орчны өгөгдөл, хэрэгслүүд, үйлдлийн урсгалыг нэг систем болгон уялдуулсан архитектурыг илэрхийлдэг \citep{anthropic-agents,react-paper}.

Оюутны туслах системийн хувьд агент нь хэрэглэгчийн хүсэлтийн зорилгыг ангилах, аль системээс ямар мэдээлэл авахыг шийдэх, шаардлагатай хэрэгслүүдийг зөв дарааллаар дуудах, олж авсан мэдээллийг нэгтгэн ойлгомжтой тайлбарлах, мөн эрсдэлтэй бичих үйлдэл дээр баталгаажуулалт шаардах чадвартай байх ёстой. Өөрөөр хэлбэл SISi системийн ухаалаг агент нь зөвхөн ``асуултад хариулдаг чатбот'' биш, харин ``өгөгдөлтэй ажилладаг туслах'' байх ёстой.

\section{Model Context Protocol (MCP)}

Model Context Protocol нь хэлний загварт гадаад хэрэгсэл болон өгөгдлийн эх үүсвэрийг стандартчилсан аргаар холбох зорилготой нээлттэй протокол юм \citep{mcp-intro}. MCP-ийн гол санаа нь загвар өөрөө бүх интеграцыг шууд мэдэх бус, тусдаа серверүүдээр дамжуулан \textit{tool}, \textit{resource}, \textit{prompt} зэрэг нэгжээр системтэй харилцах боломжтой болгох явдал юм.

Энэхүү дипломын ажилд MCP-ийг сонгосон шалтгаан нь стандарт интерфейс, модульчлагдсан бүтэц, өргөтгөх боломж, мөн аюулгүй байдлын удирдлага гэсэн дөрвөн үндсэн шалтгаантай. SISI, багшийн портал, календарь, баримт боловсруулах үйлчилгээ зэрэг өөр өөр системийг ижил загвараар илэрхийлж болох, порталтай холбогдох логик, Chrome extension-д суурилсан хэрэглэгчийн оролт, ирээдүйн интеграцуудыг тусдаа давхаргад хөгжүүлж болох, шинэ хэрэгсэл нэмэхэд бүх системийг дахин бичих шаардлага багасах, read-only болон mutation хэрэгслүүдийг тусад нь загварчлах боломжтой зэрэг давуу талууд MCP сонгох шалтгаан болсон.

Одоогийн хэрэгжилтэд локал MCP сервер нь SISI API-г оюутны сессээр баталгаажуулж, хуваарь, академик үзүүлэлт, санхүү, мэдэгдэл, e-journal, баримт бичгийн төрлийн өгөгдлийг read-only хэрэгсэл болон raw resource хэлбэрээр ил гаргаж байна. Энэ нь дипломын ажлын цаашдын агент урсгалын үндсэн технологийн тулгуур юм.

\section{Судалгааны дүгнэлт}

Онолын болон харьцуулсан судалгааны үр дүнд дараах дүгнэлтэд хүрэв.

\begin{enumerate}
  \item SISi системийн ухаалаг агент төрлийн системд дан ганц LLM хангалтгүй бөгөөд бодит системтэй холбогдох агент архитектур шаардлагатай.
  \item MCP нь тус төслийн хувьд интеграцын давхаргыг стандартчилах, өргөтгөх, аюулгүй удирдах хамгийн тохиромжтой шийдлийн нэг байна.
  \item Ижил төстэй системүүд байдаг боловч NUM-ийн SISI порталын хэрэглээ, өгөгдөл, workflow-д чиглэсэн тусгай шийдэл хэрэгтэй.
  \item Судалгааны дараагийн гол алхам нь архитектурын зөв сонголтыг бодит хэрэгжилт болон хэрэглэгчийн үнэлгээгээр баталгаажуулах явдал юм.
\end{enumerate}
