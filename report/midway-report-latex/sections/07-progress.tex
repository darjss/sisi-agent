\section{MCP серверийн хэрэгжилт}

MCP сервер нь төслийн гол техникийн амжилт юм. Одоогийн байдлаар найман read-only хэрэгсэл бүрэн хэрэгжсэн бөгөөд эдгээр нь SISI порталын үндсэн домэйнуудыг хамардаг.

Дараах найман хэрэгсэл амжилттай хэрэгжсэн. \texttt{get\_student\_overview} нь оюутны ерөнхий мэдээлэл, зураг, статус авдаг. \texttt{get\_schedule} нь хичээлийн болон шалгалтын хуваарь авдаг. \texttt{get\_academic\_record} нь үзсэн хичээлүүд, дүн, голч авдаг. \texttt{get\_financial\_overview} нь төлбөрийн мэдээлэл, үлдэгдэл, төлөлтийн түүх авдаг. \texttt{get\_notifications} нь системийн мэдэгдлүүд авдаг. \texttt{get\_student\_resources} нь сургуулийн нөөц, холбоосууд авдаг. \texttt{browse\_top\_menus} нь порталын үндсэн цэсийн бүтэц авдаг. \texttt{get\_ejournal\_summary} нь цахим сэтгүүлийн товч мэдээлэл авдаг.

Эдгээр хэрэгслүүд нь Zod schema ашиглан оролт гаралтын бүтцийг тодорхойлсон бөгөөд SISI-ийн таван API endpoint-тэй холбогддог: auth.num.edu.mn, support2.num.edu.mn, tree.num.edu.mn, phost.num.edu.mn, teach-sisi.num.edu.mn.

Session удирдлагын систем нь гурван endpoint-оор ажилладаг. \texttt{/session/register} нь шинэ session бүртгэж, access token хадгалдаг. \texttt{/session/refresh} нь session-ийн хүчинтэй хугацааг сунгадаг. \texttt{/session/clear} нь session устгаж, logout хийдэг. Session нь idle TTL (Time To Live) ашиглан автоматаар дуусдаг бөгөөд энэ нь аюулгүй байдлыг хангахын зэрэгцээ хэрэглэгчийг дахин нэвтрүүлэхгүй байх боломжийг олгодог.

\section{Chrome Extension-ийн хэрэгжилт}

Chrome extension-ийн хэрэглэгчийн интерфейс бүрэн хийгдсэн. WXT framework болон React ашиглан popup дотор чат интерфейс бүтээсэн. Интерфейс нь мессеж бичих, илгээх үндсэн функц, AI хариултыг харуулах талбар, харилцааны түүх харуулах боломж, SISI порталын брэндинг дизайн зэрэг боломжуудтай.

Гэхдээ extension нь одоогоор backend-тэй холбогдоогүй байна. Чат интерфейс нь mock өгөгдөл ашиглан ажиллаж байгаа бөгөөд бодит AI хариулт авахгүй байна. Энэ холболтыг дараагийн шатанд хийх шаардлагатай.

\section{Завсрын шатны гол үр дүн}

Одоогийн байдлаар хэд хэдэн бодит ахиц гарсан. SISI API-д холбогдох MCP архитектур амжилттай ажиллаж байгаа нь tool-augmented AI agent бүтээх боломжтойг нотолж байна. Оюутны өгөгдлийн найман чухал домэйныг read-only хэрэгсэл хэлбэрт оруулсан нь оюутны үндсэн хэрэглээний тохиолдлуудыг хамарч байна. Chrome extension-ийн UI бэлэн болсон нь хэрэглэгчийн туршлагыг турших боломжийг олгож байна. Monorepo архитектур нь цаашдын интеграц, өргөтгөлд бэлэн суурь болж байна.

\section{Үлдсэн ажлууд}

Дараагийн шатанд хэд хэдэн ажлыг тэргүүн ээлжид хийх шаардлагатай байна.

Дунд хугацааны ажлуудын хувьд AI-аас MCP хэрэгслүүдийг бодитоор дуудах orchestration урсгалыг нэгтгэх шаардлагатай. Vercel AI SDK-ийн tool calling функцийг MCP SDK-тэй холбох хэрэгтэй. Chrome extension-оос AI рүү хүсэлт илгээх, access token дамжуулах логик хэрэгжүүлэх шаардлагатай. Extension-оос авсан SISI session token-ийг MCP серверт дамжуулах, дахин нэвтрэлтгүй session bootstrap хийх урсгалыг тогтворжуулах хэрэгтэй.

Урт хугацааны ажлуудын хувьд календарь (Google Calendar), баримт бичиг (Google Drive, Notion) зэрэг гадаад үйлчилгээнүүдтэй холбогдох ерөнхий MCP хэрэгслүүд нэмэх шаардлагатай. Багшийн талын API-тай холбогдох MCP давхарга төлөвлөж, хэрэгжүүлэх хэрэгтэй. Олон оюутны дүнг Excel файлаас системд оруулах workflow хэрэгжүүлэхдээ мөр бүрийн шалгалт, баталгаажуулалт, алдааны мэдээлэл өгөх шаардлагатай. Бичих үйлдлүүдэд confirmation flow хэрэгжүүлэх хэрэгтэй. Бодит оюутнуудтай туршилт хийж, системийн үнэлгээ авах шаардлагатай.

\section{Үнэлгээний төлөвлөгөө}

Эцсийн шатанд системийг хэд хэдэн хэмжүүрээр үнэлэхээр төлөвлөж байна.

Техникийн хэмжүүрүүдийн хувьд даалгавар амжилттай гүйцэтгэсэн хувийг хэмжинэ. Өгсөн use case бүр дээр систем хэрэглэгчийн хүсэлтийг амжилттай гүйцэтгэсэн эсэхийг шалгана. Энгийн асуулт (нэг хэрэгсэл) болон олон алхамт хүсэлт (олон хэрэгсэл) тус бүрийн хариу өгөх дундаж хугацааг хэмжинэ. AI загвар хэрэглэгчийн хүсэлтэд тохирох MCP хэрэгслийг зөв сонгосон эсэхийг шалгана. Буруу хариулт өгсөн, алдаа гарсан тохиолдлын хувийг бүртгэнэ.

Хэрэглэгчийн туршлагын хэмжүүрүүдийн хувьд богино асуулга ашиглан хэрэглэгчийн сэтгэл ханамжийг 1--5 оноогоор үнэлнэ. Хэрэглэгчдээс нээлттэй асуултаар санал хүсэлт авна. Хэрэглэгчид ямар төрлийн асуулт ихэвчлэн асуудаг, аль хэрэгсэл их дуудагддаг зэргийг хянана.

Аюулгүй байдлын хэмжүүрүүдийн хувьд mutation үйлдэл дээр баталгаажуулалт зөв ажилласан эсэхийг шалгана. Token-ийн хугацаа дууссан үед зөв ажиллаж байгаа эсэхийг турших хэрэгтэй.

Эдгээр хэмжүүр нь дипломын ажлын эцсийн шатанд зөвхөн техникийн demo бус, бодит хэрэглээний үнэ цэнийг нотлоход чухал үзүүлэлт болно.
