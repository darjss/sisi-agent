\chapter{ҮНДЭСЛЭЛ БА АСУУДЛЫН ТОДОРХОЙЛОЛТ}

Их сургуулийн оюутнууд хуваарь, дүн, мэдэгдэл, төлбөр, баримт бичиг, семестрийн мэдээлэл зэрэг олон төрлийн өгөгдлийг SISI портал ашиглан авдаг. Гэвч энэ мэдээллүүд тусдаа цэсэнд тархай бутархай байрласнаас оюутан хэрэгтэй мэдээллээ олохын тулд олон дэлгэц дамжих, ижил төрлийн үйлдлүүдийг дахин давтах шаардлагатай болдог. Ялангуяа хичээлийн болон шалгалтын хуваариа тогтмол шалгах, даалгавар, мэдэгдлийг хуанли руу гараар оруулах, дүн болон кредиттэй холбоотой мэдээллийг нэгтгэн харах, мөн submit хийх, файл бэлтгэх, export хийхдээ олон системийн хооронд өгөгдөл дамжуулах зэрэг ажлуудад гар ажиллагаа өндөр байдаг.

Эдгээр асуудал нь зөвхөн интерфейсийн төвөгтэй байдлаас бус, тухайн өгөгдлийг ``оюутны зорилго'' хэлбэрээр ойлгож өгдөг ухаалаг давхарга байхгүйтэй холбоотой. Жишээлбэл, ``миний энэ долоо хоногийн хуваарийг календарьт оруул'', ``миний үзээгүй хичээлүүдийг харуул'', ``дүнгүүдийг CSV хэлбэрээр экспортол'' гэх мэт хүсэлтүүдийг одоогийн порталд шууд гүйцэтгэх товч байдаггүй.

Иймээс энэхүү төслийн үндэслэл нь SISI системийн мэдээллийг байгалийн хэлний интерфейсээр дамжуулж, шаардлагатай үед порталын API болон бусад MCP нийцтэй системүүдтэй уялдан ажилладаг туслах давхарга бий болгох явдал юм. Энэ нь оюутанд зориулсан хэрэглээний босгыг бууруулж, өдөр тутмын жижиг боловч их давтамжтай ажлуудыг хэмнэж, сургуулийн системийн өгөгдлийг илүү ойлгомжтой, хэрэглэгч төвтэй хэлбэрт шилжүүлэх, мөн цаашид илүү өргөн хүрээний оюутны үйлчилгээний платформ хөгжүүлэх суурь болно.

\section{Асуудлын томьёолол}

Энэхүү судалгааны хүрээнд дараах гол асуудлыг томьёолж байна: SISI портал болон цаашид холбогдох ижил төрлийн системүүдийн өгөгдөл, үйлчилгээг найдвартай, аюулгүй, өргөтгөх боломжтой хэлбэрээр Chrome extension-д суурилсан байгалийн хэлний интерфейстэй хэрхэн холбож, хэрэглэгчийн хүсэлтийг бодит системийн үйлдэл болгон хөрвүүлэх вэ?

Энэ асуудлыг шийдэхэд зөвхөн хэлний интерфейсийн өнгөц давхарга хангалтгүй учир систем бодит цагийн өгөгдөл ашиглах, хэрэглэгчийг дахин нэвтрүүлэхгүйгээр одоогийн access token, сессийг аюулгүй ашиглах, унших үйлдэл болон бичих үйлдлийг ялгаж удирдах, олон системийн хооронд уялдаа хэрэгтэй, мөн оюутны болон багшийн өөр өөр workflow-д тааруулах боломжтой модульчлагдсан бүтэц шаардлагатай.

Тиймээс энэхүү дипломын ажлын судалгааны гол чиглэл нь агентын онол, MCP стандартыг ашигласан архитектур, мөн бодит интеграцын түвшний системийн загварчлал болно.
