\section{Технологийн сонголтын тайлбар}

Технологийн стек сонгохдоо нэг хэл дээр олон давхаргыг хэрэгжүүлэх, MCP болон орчин үеийн AI integration-тай нийцэх, prototype-оос production түвшин рүү өсгөх боломжтой байх, мөн семестрийн хугацаанд хөгжүүлэлт хийхэд практик байх зэрэг шалгуурыг харгалзсан.

Энэхүү сонголт нь дипломын ажлын хувьд оновчтой. Учир нь төслийн төвд ``agent + integrations'' архитектур байгаа бөгөөд олон жижиг үйлчилгээ хооронд ойлгомжтой интерфейс тогтоох нь цэвэр фронтенд эсвэл цэвэр backend төслөөс илүү чухал юм.

\section{Цаашид нэмэгдэх технологи}

Дараагийн шатанд гадаад MCP үйлчилгээ болон байгууллагын бусад API-тай холбогдох зөвшөөрлийн урсгал, tool calling-ийн баталгаажуулалт, approval layer, багшийн порталын mutation workflow болон Excel file ingestion, export болон submit төрлийн mutation үйлдлийн лог, мөн туршилт, үнэлгээний нэмэлт сангийн бүтэц зэрэг технологийн хэсгүүд нэмэгдэх төлөвтэй байна.

Эдгээр өргөтгөл нь одоогийн суурь архитектуртай зөрчилдөхгүй, харин модульчлагдсан хэлбэрээр дээр нь нэмэгдэхээр төлөвлөсөн.
