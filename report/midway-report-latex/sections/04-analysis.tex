\section{Функциональ шаардлага}

Системийн функциональ шаардлагыг дараах байдлаар тодорхойлов. Хэрэглэгч Chrome extension-оор дамжуулан байгалийн хэлээр асуулт, даалгавар өгөх боломжтой, систем оюутны идэвхтэй сесс дээр тулгуурлан SISI API-гаас бодит өгөгдөл унших чадвартай, хэрэглэгчийг дахин нэвтрүүлэхгүйгээр тухайн порталаас авсан access token эсвэл холбогдох сессийн мэдээллийг аюулгүй бүртгэдэг, хуваарь, дүн, санхүү, мэдэгдэл, баримт бичиг, e-journal зэрэг үндсэн домэйнүүдийг тусдаа хэрэгслээр дуудах боломжтой, оюутны өгөгдлийг ойлгомжтой текст хэлбэрээр нэгтгэн хариулдаг, ирээдүйд дурын MCP нийцтэй гаднын үйлчилгээ болон багшийн порталын API-тай холбогдох өргөтгөх чадвартай, экспорт, submit зэрэг mutation үйлдэлд баталгаажуулалт шаардах механизмтай, хэрэглэгчийн session-id, хугацаа дуусах төлөв, зөвшөөрөгдсэн origin-уудыг удирддаг, мөн хэрэгслийн үр дүнг structured content болон raw resource хэлбэрээр хадгалж, тайлбарладаг байх шаардлагатай.

\section{Функциональ бус шаардлага}

Систем нь нэвтрэх мэдээллийг ил хадгалахгүй, локал сессийн хугацааг хязгаарлаж, зөвшөөрөгдсөн origin ашиглах зэргээр аюулгүй байдлыг хангах шаардлагатай. Мөн шинэ MCP домэйн, шинэ гадаад интеграц, шинэ client төрлүүдийг бага өөрчлөлтөөр нэмэх өргөтгөх чадвартай, upstream API алдаа гарсан тохиолдолд ойлгомжтой алдаа буцаах найдвартай байдлаар ажиллах, энгийн унших хүсэлтүүдийг боломжит богино хугацаанд гүйцэтгэх гүйцэтгэл сайтай, аль хэрэгсэл ямар өгөгдөл ашигласныг тайлбарлах хянах боломжтой, мөн порталд аль хэдийн нэвтэрсэн хэрэглэгч нэмэлт нэвтрэлтгүй ашиглахад ойлгомжтой байх шаардлагатай.

\section{Эрсдэл ба хязгаарлалт}

Завсрын шатанд SISI порталын API бүтэц албан ёсны тогтвортой public contract биш байж болох тул өөрчлөлт гарч болох, бичих үйлдэлтэй холбоотой submit, export, state change төрлийн функцуудыг шууд нээх нь аюулгүй байдлын нэмэлт шалгалт шаарддаг, хиймэл оюуны загварын тайлбарлал буруу байвал буруу хэрэгсэл дуудах эрсдэлтэй тул tool routing болон баталгаажуулалтын давхарга чухал, олон төрлийн MCP үйлчилгээ болон багшийн порталын mutation урсгал нэмэгдэхэд зөвшөөрөл, аудит, алдаа шалгалтын төвөгшил өсөх зэрэг эрсдэл, хязгаарлалтууд ажиглагдаж байна.
